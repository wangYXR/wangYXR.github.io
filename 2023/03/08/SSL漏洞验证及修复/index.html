CVE-2016-2183

漏洞影响版本：

  • OpenSSL Project OpenSSL < 1.1.0a
  • OpenSSL Project OpenSSL < 1.0.2i
  • OpenSSL Project OpenSSL < 1.0.1u

漏洞产生原因：

ssl的配置中使用了3DES加密算法，这种加密算法现在一已经不在安全。

漏洞验证方法：

使用nmap对目标进行探测：

nmap -sV –script ssl-enum-ciphers -p <端口>

漏洞修复方法：

apache

修改ssl.conf文件中SSLCipherSuite项中的3DES选项，在前面加！号后保存，重启服务器。

Nginx:

需要修改nginx.conf文件中的内容

从新编译nginx后安装

windows 远程桌面

须关闭如下加密：

1、通过cmd或运行打开gpedit.msc

2、依次打开：管理模板>网络>SSL配置设置

打开ssl 密码套件顺序，选择启用并修改选项：

将选项内的内容复制到text中删除出现的3DES、DES、RC4等项：

将修改好的内容替换原来选项内的内容，应用选项

重启服务器
